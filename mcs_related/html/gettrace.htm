Function synopsis

GETTRACE - plots traces from the original MCRacks datafile

Function comments

 
% [h, rawTRACES, fitTRACES] = gettrace(varargin). 
% <h>          handles to the traces in the plot 
% <fitTRACES>  filtered data 
% <rawTRACES>  raw data 
% 
% <varargin>   can be parameter/value pairs: 
% 'subpopn'    refers to a subpopulation value created during 
%              spike sorting (integer) 
% 'datatype'   'electrodename' (default), 'cid' or 'pointer' 
% 'datafile'   location and name of the MCRackfile (default is 
%              RACKSETUP.sourcefilename) 
% 'plottrace'  'yes' to plot the trace, default is 'no' 
% 'dataset'    can be a string giving an electrode name or a list of 
%              pointers to the onset of the traces in the original 
%              data file. <dataset> cannot be a ChannelID!. 
% 'tracetype'  can be 'raw' (default) or 'filtered', in which case 
%              Savitzky-Golay filtered traces will be displayed. 
% 
% U. Egert 1/98
