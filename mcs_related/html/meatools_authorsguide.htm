GUIDE FOR AUTHORS

 

 

Add new routines to the menues

-        write the routine as a script or GUI supported and copy all necessary files into the MEA-Tools folder
or anotherone within the MATLAB path,

-        open the file AddPersonalMeaToolsMenu.m,

-        follow the instructions there to select the main menu in which to include the new routine,

-        scripts can be opened for editing by typing 'edit yourfilename' as a callback

-        adjust the menu label and callback lines as required, it is often useful to include a tag as well,

-        uncomment the last line in the meatoolsbox.m file,

-        restart the MEA-Tools from the command line.

 

A brief example:

Assume you wrote a GUI assisting in the analysis of paired pulse facilitation in brain slices. You would have to retrieve and analyze the individual triggered sweeps of the LFP response using the LFP-Tools/Extract LFP Features. After displaying, let's say the values of the response amplitude you would find those values in a matrix, named outmat per default. Paired pulse facilitation of course refers to two responses to two trigger events, so outmat would in our case be 64xNx2, where N is the number of trials (one per sweep). Let us further assume that you varied the stimulus strength to see how that affects PPF. For each setting 5 sweeps were recorded. You will probably want to average across those 5 sweeps, summarize the resulting values and calculate the ratio between the amplitudes of the response to the first and second stimulus.

In fact, such a GUI is already included in the MEA-Tools. You'll find it in the LFP section as the last menu item. Open AddPersonalMeaToolsMenu.m to see how it was included in the menu.

 

 

General jobs to do to adjust meatools routines for new MEA-Types or data acquisition systems.

 

To access different file formats: 

-        write a new 'datastream.m' function that creates the datastream object from your file format. Information about the properties, i.e. fields, to supply, can be found in dstreaminfo1.htm,

-        adjust the nextsweep.m and nextdata.m functions such that they retrieve the data from your binary files correctly. If the MEA has a fixed geometry, nextdata.m needs to place data from one electrode into one row in the sequence described below.

-        It is not necessary that this uses mex-files or DLLs. Any version that retrieves your data and supplies the necessary information is fine.

 

To use different MEA layouts:

Standard variables: MEAINFO

-        load meainfo.mat - add the fieldnames matching your MEA-type descriptor:

 

add the following fields:

-        MEAINFO.<meatype>cid : vector with integer identifiers for the electrodes of your MEA geometry. This needs to use the addressing scheme Matlab uses for linear indices into a matrix with the corresponding layout. These channelIDs point to rows in data matrices recorded by the electrode identified with the same channelID used as a linear index into the MEA matrix.

 

Example:

if the MEA has an 8x8 layout the top row ChannelIDs would be 1,9,17,25,33,41,49,57. A full dataset has 64 rows.

for an 10x6 layout these would be 1,11,21,31,41,51. A full dataset has 60 rows. Also see the LFP-Tools section in the MEA-Tools/Documentation/Overview file

 

-        MEAINFO.<meatype>names : matrix with string identifiers of the MEA electrodes according in the same sequence as the channelIds above

 

 

other m-files to adjust:

- add new MEA-type descriptor defining the number of rows and columns according to the description given there

in trace2mat.m; subfunction 'getcutouts'

- adjust for your file type
