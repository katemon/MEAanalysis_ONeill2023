Function synopsis

readspike2data - reads exported SPIKE 2 data into MEATools compatible format

Function comments

 
% [CNspk2] = READSPIKE2DATA(fname) with the file <fname> being an export file from SPIKE 2 
% reads the waveforms, channelnumber, clusternumber (if applicable), and spike time of each 
% channel and spike. The data are stored in global variables compatible with MEATools 
% spike data analysis: 
% 
% RACKSETUP, SOURCECHANNELNUMBER, SPIKETIME, PARSESETUP, spkpointer, TRACES, PCVECTORS. 
% 
% <CNspk2> holds the unit IDs assigned by SPIKE 2 spike sorting. It is compatible 
% with MEATools if copied to CLUSTERNUMBER (global), so you can compare the different 
% sorters. 
% 
% <TRACES> is a matrix of spike cutouts with one column per spike. The traces are 
% padded with zeros to account for unequal cutout length, which may or may not 
% affect the principle components calculated. 
% 
% NOTE: since the electrode/channel identifiers of SPIKE 2 are not compatible with the 
% notation used with MEAs a wrong electrode number will be assigned. This is necessary 
% to ensure compatibility of the various display tools. From any channel name listed 
% in one of the GUI fields of the MEATools you may retrieve the original channel 
% identifier using the command -- getcid(<electrodename>) --, in which <electrodename> 
% is a string with the name displayed, e.g. getcid('# 23') will return 11 as channel ID. 
% 
% 
% (c) U. Egert 2/99 
% 
% See also WRITECN2SPIKE2
