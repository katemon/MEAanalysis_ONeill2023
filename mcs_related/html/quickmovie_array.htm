Function synopsis

quickmovie_array - quick & dirty tool to generate a movie from a 64xN matrix

Function comments

 
% data = quickmovie_array(varname, startend, varargin) 
% <DuV>              a 64xN array, a sampling period of 0.04 ms (25kHz sampling rate) 
%                    is assumed unless you state otherwise with a parameter/value 
%                    pair. e.g. ..., 'samplingperiod', 0.1, ... for 10 kHz. 
% <startend>         2-element vector giving start and end times for the movie in msec 
% <data>             the data actually used for the movie 
% 
% <varargin>         several optional parameter/value pairs (with their defaults): 
% skip = 5;          no. of samples skipped between frames 
% ipfactor = 2;      interpolation depth 
% ipMethod = 'cubic' interpolation method ('linear', 'cubic') 
% preIp = 'off'      pre-interpolation, 'on' delays the start but speeds up the movie 
% viewDim = '2D'     '2D' or '3D' display 
% loop = 'on'        infinite looping through the movie 
% zlim = []          z-axis and color limits in µV, def. are min and max in the dataset 
% integrate = 'no'   'yes' to show the inverted, detrended integral of the data 
% plot8x8 = 'yes',    'yes' to generate an 8x8 figure of the data (calls plot64.m) 
% saveas = [];       path and group name for jpeg-files, e.g. 'saveAs', 'c:\temp\test'. 
%                    If 'saveAs' is a number (any) a global matlab-movie variable named 'mframe' 
%                    will be created from the movie. 
% namesequence='sequential'; Indicates the numbering of the filenames given by 'saveAs'. 'sequential' 
%                    will have ascending numbers appended to the filename. 'frameindex' will have 
%                    the framenumber appended instead. 
% 'position'         size and position of the movie figure relative to the screen (0 is left or bottom 
%                    of the screen, 1 is top or right) 
%
