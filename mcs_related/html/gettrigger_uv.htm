Function synopsis

gettrigger_uV - retrieves MC-Rack continuous data as a matrix of uV values

Function comments

 
% currOutVar=gettrigger_uV(D, newseTime) 
%     <D> must be the name of a datastream object or a 
%     file of continuous raw data recorded with MCRacks. 
%     <newseTime> is a 2-element vector giving the start and end 
%     time in milliseconds. 
%     <currOutVar> is a variable of voltage data in uV. 
% 
% If called without arguments an input dialog opens, requesting the 
% necessary information, suggesting defaults based on the previous 
% call. 
% 
% Parameter value pairs may be used to select specific trigger StreamNames. 
% 
% If trigger events are queried from more than one trigger stream, the 
% return variables will be cell arrays. With one trigger stream only the 
% return variables will be doubles/strings! In other words, the class of 
% the return variables matches that of the variable used to list the 
% trigger datastreams. 
% 
% example 
% ..., 'StreamName', {'Trigger 1', 'Trigger 2'}); 
% 
% This will return a 2xN cell array matrix with the data from Trigger 1 and Trigger 2 
% in row 1 and 2 respectively. 
% 
% Note: 
% The most recent settings will be stored with set(0, 'userdata') and reused 
% as defaults in the next call to getraw_uv via the dialog box, i.e. without input 
% arguments. 
% 
% U. Egert 3/99
