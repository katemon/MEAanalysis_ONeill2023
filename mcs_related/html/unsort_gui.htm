% unsort_gui(cfginfo, datainfo) evaluates edit fields as 
%    indicated in <cfginfo> and handles either MCS data style, or 
%    other formats if <datainfo > is supplied. Both input arguments 
%    must be structures with the following fields. It is used in the 
%    spikesortbox GUI. 
% 
% cfginfo.fighandle      a handle to the figure whos edit fields should be 
%                        used 
% cfginfo.tagname        the tag label of the edit fields that should be 
%                        searched for the userdata{1} cell with the strings 
%                        'subpopulation', 'ename', and 'newnum' 
% 
% datainfo.clustervar    the name of the variables and indices that should 
%                        be used for the ID of the clusters as a string, e.g. 
%                        if the 3rd column of an array DATA would be the 
%                        clusternumbers you would write 'DATA(:,3)' 
% datainfo.scnvar        the name of the variables and indices that should 
%                        be used for the ID of the channels/electrode names, 
%                        as a string, e.g. with gdf type arrays this would be 
%                        'gdf(:,1)' 
% datainfo.datavars      a cell array with variable names that must be global 
%                        in the workspace and need to be used for this 
%                        function, e.g. {'CHANNELS', 'CLUSTERS'}. This saves 
%                        memory. 
% 
% NOTE that the variables for channel number and cluster number must be sorted 
%         in the same order. 
% 
% U. Egert 
% 
% See also UNSORT
