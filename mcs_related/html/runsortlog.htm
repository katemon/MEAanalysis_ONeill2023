Function synopsis

RUNSORTLOG - applies spike prepared sorting settings on a datasets

Function comments

 
% runsortlog(sortparams, cn) is intended to apply spike sorting parameters 
%    worked out with one set of spike data to automatically sort a second 
%    dataset. It requires a <sortparams> variable with the necessary 
%    parameters as described in the help for readsortlog.m and writesortlog.m. 
%    These functions can be used to generate, store and retrieve a 
%    parameter file with parameter settings. writesortlog.m is called during 
%    sorting with the Sort Spikes menu in MEA-Tools. 
%    <cn> is optional and can be used to restrict the sorting to a particular 
%    CLUSTERNUMBER. 
% 
%
