% This tools allows you to view the results obtained by LFP Analysis I, 
% it relies on it and you therefore need to select the variables that 
% you stored those results in. See the tooltip strings for further help. 
% 
% The content of the data variable must be generated by lfp_extract, 
% e.g. with the lfp_extractgui. This is a structure array holding 
% fields for the various extracted parameters with one row per electrode, 
% one column per sweep, and one plane per trigger. 
% 
% The results variable will have as many rows as there are channels in <data>, 
%    a column for each sweep, and a plane for each trigger. 
% 
% !! NOTE that values for missing traces are set to 0 !!!! 
% 
%-------------------------------------------------------------------------- 
% Parameter selection 
% 'min' / 'max'         the absolute minimum/maximum in the 
%                             evaluation range of the traces, 
% 'amplitude'           minium to maximum voltage. Amplitude will be 
%                       negative if the maximum preceeds the minimum, 
%                       positive otherwise. 
% 'area'                total area in the evaluation range (sum of 
%                       absolute sample values) 
% 'poly-min/poly-max    minima and maxima of a polynomial fit to the data 
%                       (currently disabled) 
% 
%  Parameter type 
%  'voltage             the voltage corresponding to the above selection 
%  'time'               the post trigger time corresp. to the above 
% 
% Display style         values for each electrode shown as 
% 'trace'               one common line plot 
% '8x8 line plots'      8x8 lines in separate subplots 
% '8x8 bar histo'       8x8 histograms in separate subplots 
% 'pseudocolor'         64xN image plot (pseudocolor) 
% 'movie'               interpolated movie (opens movie GUI). 
% 
% x-axis type 
%  'sweep ID'           values shown vs. sweep ID on the X-axis 
%  'time'               dito shown vs. sweep start time 
% 
% Display time units 
%  s/ms/micros         time scale to use for the x-axis 
% 
% Data range limits 
%  'all'                show all values 
% [m n]                 show values between sweep m and n (can 
%                       be imitated by zooming 
% Sweep times variable  name of the variable holding the sweep start time 
% Sweep ID variable     name of the variable holding the sweep IDs 
% 
% Trigger Index         index to the trigger within a sweep that should be 
%                       displayed 
% 
% Results variable name name of the base-workspace variable that will hold 
%                       the displayed data set. 
% 
% Options menu 
%  'save settings to file'    save the current GUI to a mat-file 
%  'load settings from file'  corresponding load function 
%  'save configuration to userdata 
%                             save the configuration as a structure 
%                             holding all necessary information to 
%                             the userdata property of the GUI, this 
%                             is occasionally useful to link other 
%                             tools. 
%  'save configuration to the base workspace 
%                             likewise but saving the structure to 
%                             the workspace. 
%
