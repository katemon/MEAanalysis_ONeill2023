SORTHELP - help for the sorting tool GUI started with 'spikesortbox'

Function comments

 
% 
% General information 
%--------------------- 
% This GUI is used to sort spike data based on spike waveforms cut 
% beforehand from MCRacks spike data records. It relies on a number of 
% prerequisites that are given only if the data were recorded with this 
% software and parsed offline with rdspx86.m. Sorting is based on the 
% contributions of principle component vectors calculated in this function. 
% 
% The following global variables are used in the base workspace and must 
% present when the GUI is started, i.e. before you can call this help.: 
% 
% SPIKETIME                a col. vector of spiketimes (microsec. if 
%                             MCRacks was used) 
% SOURCECHANNELNUMBER      a col. vector of channel numbers (ChannelIDs 
%                             listed in RACKSETUP) 
% PARMAT                   a 2D-matrix with one row of PCA weights per spike 
% RACKSETUP                a structure describing the recording setup 
%                             and the original data 
% PARSESETUP               a structure listing data established during 
%                             analysis of the raw data 
% spkpointer               a col. vector pointing to the beginning of the 
%                             spk-waveform data in the raw data file (older 
%                             versions of MCRack only). 
% 
% Spike sorting will assign a CLUSTERNUMBER to each spike, resulting in a 
% a col. vector of length(SPIKETIME). Additional information on the 
% resulting cluster assignments is stored in PARSESETUP.Clusternames and 
% PARSESETUP.Clusternames_def. 
% 
% 
% Checklist to optimize data validity and integrity in MEA recordings from slices 
%--------------------- 
% - Reduce EMF interference as much as possible 
% - minimal length of  the spike cutouts should be 2+3 ms 
% - filter data w/ savgol 
% - verify the homogeneity of the spikes in a cluster 
% - set CLUSTERNUMBERS of any dubious spikes to 0 
% 
% 
% 
% Information on usage 
%--------------------- 
% 
% 'location of the source data file': path and filename of the original 
% MCRack file. During sorting spike waveforms will be retrieved from 
% this file. They are not stored in the matlab workspace permanently. 
% 
% <electrode name> and <subpopulation> are selected from drop-down lists. 
% The <subpopulation> list is updated only after selecting an electrode. 
% Default is clusternumber 0. 
% 
% 'sort parameters' can be selected from a list. 
% 
% 'target file': the mat-file stored with the 'save workspace button'. 
% 
% <save workspace> will save ALL variables in the base workspace to the 
% current target file. The 
% <close all> button will however delete temporary variables not needed 
% after sorting (e.g. some figure handles). To reduce the files size you 
% may want to close the figures first and then save your data manually on 
% the command line with 'save'. The <save workspace> button is convenient 
% to store your data once in a while during sorting. 
% 
% <show clusters> will plot all cluster assignments of the selected 
% electrode with spikes as dots colored according to their clusternumber. 
% Use the 'spike record display' menu to review the spike cutouts in their 
% temporal order with clusters color coded. 
% 
% The interface guides you through the sorting processes with a button 
% list in a figure named 'sortgui'. Additional functions are available 
% via figure menus and context menus activated with the right mouse button. 
% 
% U. Egert 6/98 
%
