getparamdata - retrieve parameter data from a datastream object

% [dout, times]= getparamdata(X, paramstreamname, param, startend) 
% retrieves parameter data from an MCRack file. <X> indicates the path and 
% name of an MCRack file (*.mcd) or a datastream object corresponding to 
% such a file. <paramstreamname> must indicate a valid parameter stream 
% name. <param> indicates the parameter type requested, e.g. 'max'. 
% Multiple parameters can be queried by placing their strings in a cell 
% array of strings, e.g. {'max', 'tmax'}. <startend> is a 2-element vector 
% defining the requested time window in milliseconds. 
% 
% <dout> will be a structure with the following fields: 
% <dout>.StreamName indicating the parameter data stream, e.g. 'Parameter 1', 
% <dout>.Inputbuffer will indicate the source data stream, e.g. 'Spikes 1' 
% <dout>.<param> will hold a matrix of parameter values, e.g. 'rate'. Its size depends on the 
% number of samples (columns), the number of electrodes (rows) and the 
% setting of the 'make64' switch (see below). 
% 
% <times> is a vector of time values in ms corresponding to the sampling 
% times in <dout>.<param>. 
% 
% 
% Note that you can only retrieve the values of one <paramstreamname> at a 
% time. 
% 
% Parameter/value pairs: 
% 'make64'  == 1 places the result in a 64xN matrix, independent of the number 
%           of electrodes acutally holding data. Parameter data derived 
%           from analog channels will always be returned in the sequence 
%           and matrix size corresponding to the electrodes requested. 
%           'make64' for is ignored in this case. 
% 
% Example: retrieving spike rates from the first 100 seconds of a recording 
% [dout, times]= getparamdata('c:/myMCdata.mcd', 'Parameter 1', 'rate', [0 100e3, 'make64', 1); 
% 
% To plot such data use 
% plot_MxN(dout.rate, 'samplingperiod', times) 
% 
% (c) U. Egert
