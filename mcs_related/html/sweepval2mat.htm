sweepval2mat - parses the result variables from lfp_extract

Function comments

 
% 
% Syntax 
% outmat = sweepval2mat(data, varargin); 
% 
% <data> must be generated by lfp_extract, e.g. with the lfp_extractgui. 
% 
% <outmat> will have as many rows as there are channels in <data>, 
%    a column for each sweep, and a plane for each trigger. 
% 
% arguments to use in 'parameter', 'value' pair style are: 
% <parameter> can be: 
%              'min' / 'max'    to retrieve the absolute minimum/maximum 
%                             of the traces, 
%              'pmin' / 'pmax   to retrieve the minimum/maximum of the 
%                             polynomial interpolation of the trace 
%    'parametertype'   sets ('voltage') or 'time' for retrieval the sample-indices of <parameter> 
%    'extridx' sets the index of the extremum to be retrieved, with 'pmin'/'pmax' 
%                   only, default is 1. 
%    'nswp'    sets the number of sweeps to retrieve (not their index) 
%    'ntrig'   sets the number of trigger events within each sweep to retrieve, 
%                   data for different triggers are stored in the 3rd dimension 
%                   of <outmat>. 
% 
% !! NOTE that values for missing traces are set to 0 !!!! 
% 
% U. Egert 6/98 
% 
% See also LFP_EXTRACTGUI NEUROTOOLS LFP_EXTRACT SWEEPEVALSCRIPT SWEEPEVALPLOT
