Function synopsis

getsweep_uV - retrieves an MC-Rack sweep as a matrix of uV values

Function comments

 
% [sweep_uV, sweep_time] = getsweep_uV(sourceobj, targetsweep); 
%    <sourceobj> must be the name of a datastream object or of a 
%    file of triggered sweeps recorded with MCRacks. <targetsweep> 
%    is the index number of the sweeps to be retrieved, it can be 
%    range (e.g. 1:3), or a comma separated list of indices (e.g. 1,3,5). 
%    <sweep_uV> is a 64xNxP matrix of sampling values given in microvolts, 
%    where P is the length of <targetsweep> and N is the number of 
%    samples in a sweep. <sweep_time> is a matrix giving the experimental 
%    time in seconds with one row per sweep, e.g. for plotting. 
% 
%    If the data were recorded as 'Start And Stop On Trigger' windows the 
%    no. of samples per sweep will likely vary. <sweep_uV> and <sweep_time> 
%    in this case will be cell arrays with one cell per sweep giving uV values 
%    and a time vector (experimental time) per sweep. 
% 
% Parameter/Value pairs: 
% 'withinSweepWindow'   2-element vector defining the retrieval window WITHIN each sweep in ms (optional) 
% 
% If called without arguments an input dialog opens to request the necessary 
% information. 
% 
% See also GETANALOG_UV GETTRIGGER_UV GETRAW_UV 
% 
% (c) U. Egert 9/98
