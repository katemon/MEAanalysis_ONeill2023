MEA-Tools
MATLAB tools for the analysis of multi-neuronal data recorded with multi-electrode arrays
Version 2.8

Copyright U.Egert 1998-2004

 
 
 
MEA-Tools is a collection of MATLAB-based tools to analyze spike and field potential data from extracellular recordings with Multi-Electrode-Arrays, as produced by the NMI Reutlingen, and the MCRack (Vers. 1.4 or higher) recording software. MCRack is part of the MEA-60 System and the ME16, ME32, ME64 and ME128 recording setups distributed by multichannel*systems. MEA-Tools is meant to be an easy-to-use toolbox with graphical user interfaces (GUIs) on one hand and a script-based guide through some useful procedures on the other. These scripts can be expanded as needed to incorporate new data analysis or display features as required - and obviously depending on your knowledge of MATLAB. We will incorporate some scripts into GUIs and add additional features ourselves as our own work progresses.

The Toolbox is compatible with MCRack recording obtained with the ME-16 to ME128 systems that do not use planar electrode arrays. Obviously, display functions that rely on a known geometric arrangement of the electrodes on a rectangular grid are not useful with generally loosely defined positions of wire electrode bundles. We currently do not catch all errors that may occur if you use these functions erroneously. Which may cause unpredictable reactions by matlab, including the notorious 'segmentation error'. So make sure you save your data before you experiment with new functions.

Generally, MEATools is arranged in 4 menus for spike analysis, LFP analysis, continuous data tools and common display functions respectively. Descriptions and help for the individual m-files used in this suite are available through MATLAB help functions or HTML-documentation, which will also show the cross-references between the files.

Several other functions and utilities are not used in the context of any GUI but are useful for command line work, e.g. quickmovie.m. Several commands will come up with a simple GUI when they are called without input arguments, e.g. getraw_uv.m, getsweep_uv, writemcs2neximp.m.

IMPORTANT
The central variable in the MEA-Tools context is a special data object named D of the class 'datastrm'. It holds the essential information about the recording conditions and the data file. It also links this file to your matlab workspace. If the corresponding MCRack file is not found at the previous location this connection is lost. The datastream object D is then converted to a structure, with its information still available. However, it will not be possible to retrieve waveform data from the file unless you recreate D as a datastream object with the command:  D = datastrm('filename').
 

MEA-Tools startup screen

The MEA-Tools menu consists of a number of submenus collecting functions useful for a particular data type. Display functions were collected in a separate menu since some are used for all data types.

The documentation section points to either this page, a matrix showing the crosslinking of the m-files used and a list of graphics tools that are not part of the matlab suite. Details of the sections are describe below. Please refer to the individual help menus for more details.

SPIKE TOOLS

Spike Tools are designed to load, view, sort and analyze specific aspects of electrophysiological spike data recorded with the MCRacks multi-electrode array system for 8x8 MEAs. Many features rely on this file structure, or the arrays created thereof, some can be used with any matlab-array. We will expand the flexibility to use other formats as time allows.

Spike Tools currently holds the following menu items:
'Parse Spike Files'  - read the spikes from the original data files and calculate descriptive parameters (currently principle components), save the results as mat-files. This is the first thing you need to do when working with spike data.
'Load Mat-Files'  - load such files
'calculate principle components from a matrix' - supplementary tool to work with simple matrices holding spike cut-outs
'define/edit RACKSETUP' - assists in defining RACKSETUP (see below) when working with data sets other than MCRack files
'Spike Data Summary' - review a summary of the current dataset
'Sort Spikes' - sort the spikes according to these parameters,  this includes tools to review spike shapes. The final parameters and cluster boundaries are stored to a log file to:
'Reuse Sorting Parameters'  - a script guiding you through applying the same spike sorting sequence to a new data set, thus creating comparable cluster assignments
'Merge spike data files' - a script guiding you through the process of merging sorted data from successive recordings into a single dataset. The resulting dataset cannot access the original raw spike data files (i.e. shapes). Sorting should therefore be finished before merging.
'Spike Record Display' - a display routine showing the spike cut-outs vs. time and color-coded for their cluster assignment during spike sorting. A useful tools to review the results of your spike sorting.
'Spike Rate Analysis' - different tools to calculate and display spike rate based on moving window rate estimation.

 
 

How to proceed:

  1. First of all, the spike data, the waveforms need to be extracted from the datafiles to retrieve spike times, calculate parameters necessary for spike sorting and to review the data. The matlab data structures created during this process are currently used as global variables in many of the components and therefore can be used only if their names are not changed.

    2. The essential variables in the workspace are
 D the datastream object linking your workspace to the MCRack file
SPIKETIME spike times given in microseconds
SOURCECHANNELNUMBER  a number giving the electrode in matlab indices for 8x8 matrices.
RACKSETUP a structure holding information written to during the experiment. ChannelNames, MicrosecondsPerTick, and ChannelID lists are subfields
PARSESETUP information collected during data processing.
PARMAT a parameter matrix describing principle component contributions and other features of each spike
spkpointer a pointers into the data file for each spike
CLUSTERNUMBER during spikes sorting newly assigned electrode-subnames are collected in PARSESETUP and identifiers for spike cluster membership of each spike are listed in this variable. It does not exist prior to spike sorting.
  1. To use previously extracted datasets use 'Load Mat-Files'
  2. To review the data set in a summary use 'Spike Data Summary'
  3. sort raw data sets to isolate single neurons and remove artifacts with 'Sort Spikes'. This generates CLUSTERNUMBER and modifies PARSESETUP
  4. to reuse the cluster boundaries you just established on a new data set, e.g. from the same preparation use 'Reuse Sorting Parameters'. See MATLAB help-file for details on the requirements for successfully reusing sorting parameters.
  5. To join these disjunct sets into one use 'Merge Spike Data Files'
Once you have established your dataset you may go to the actual analysis. :
  1. Calculate and view spike rates for single or multiple electrode/subpopulation combinations as trace or pseudocolor displays
  2. calculate PSTH displays and dot-plots for triggered sweeps
  3. identify spike bursts and inter-spike-interval distributions

LFP TOOLS

LFP Tools are designed to load, view and extract specific aspects of electrophysiological data recorded with the MCRacks multi-electrode array system for 8x8 MEAs. Most  features rely on this file structure,  some can be used with matlab-arrays if they are arranged in 64xN columns and rows with one row per electrode. Rows must be arranged according to the matlab adressing scheme for an 8x8 matrix. In other words the row index links a trace with the electrodes position in an 8x8 MEA. The row index corresponds to the linear index into a matlab 8x8 matrix pointing to this electrode.

Although the tools are stable within they need to use a 'datastream object' named D. To create this object in the base workspace use the 'load file' tool. When storing a workspace Matlab will lose the datastream object identity of D and instead store it as a structure. This does not link D to the original data file anymore (the file is not open), you need to recreate D.
 

LFP Tools currently holds the following tools:

File Handling Utilties
'load sweep data' - create the datastream object D from an MCRack file
'load mat-files'  - load a Matlab workspace
'retrieve a triggered sweep' - retrieve the data of an individual sweep from a file
'average triggered sweeps' - calculate an average an the standard deviation across one or more sets of triggered sweeps of data
'manual sweep handling' - a script utility useful if you need to retrieve and manipulate data manually.

Extract LFP Features - The feature extraction tool is used to extract some characteristic values from the sweeps. Currently 'min' and 'max' values and their times  in an adjustable range of the data set  are retrieved. The same can be done for differentials of the data, for multiple trigger events, after subtracting other waveforms (e.g. stimulus artifact templates). A number of preprocessing tools is available. The tool allows to review the peaks found and batch processing of sets of sweeps. For large amounts of data this may take some time. A typical LTP experiment will be processed in a few minutes, for a long experiment you might want to run it over night.
Display LFP Features - A tool to view the data extracted in by the previous function with various displays, static and animated. This also summarizes the data display in simple matrices that can be used for further processing.

LFP Statistics - a script guiding you through calculating averages and standard deviations across sweeps.


CONTINUOUS DATA TOOLS

The tools to retrieve and work with continuous records of the electrode potentials a currently restricted to just retrieving a slice of data to a variable in the base workspace. The needs for further processing vary considerably so standard tools that would be useful for some number of people cannot be defined.easily. Any user contributions to this section are welcome. I can contribute some functions and scripts for specific tasks, such as picking spikes from a continuous record or processing data from cardiac myocyte cultures, if anybody is interested in them.
 


SUPPLEMENTARY DATA TOOLS

This section supports the retrieval of Trigger, Parameter and Analog data streams.

SUPPL.  DATA TOOLS currently holds the following tools:
'retrieve analog data from a continuous record' - retrieve a time slice of gap-free data from an 'analog data' datastream. These are recorded through BNC connectors on the back of ADC board on the back of your computer.
'retrieve a sweep of analog data' - similar, but for triggered sweeps.
'retrieve trigger data' - retrieve a 'Trigger' data stream from a file
'retrieve parameter data' - retrieve data generated by the 'Analyzer' tool of MCRack


DISPLAY FUNCTIONS

The Display Functions menu currently holds the following menu items:
Datastream Summary A tools summarizing the properties of MCRack files and their datastreams
'8x8 Subplots' Create an 8x8 subplot figure from datastream objects or 64xN matrices.
'Movie Tool' The 'movie tool' shows a MEA-geometry oriented pseudocolor movie of either the data in D, or of an 64xN array. Values are coded in color and can be viewed in 2D or 3D display with different degrees of interpolation and from various perspectives. The tool can not only be used to view raw data  is useful to visualize pre-processed data as well, e.g. response-amplitudes across trials or PSTH data from MEA records.Several preprocessing options are available. The movie uses matlab as its engine and currently cannot be saved as a standalone file. However, a series of numbered jpeg files can be saved. Corel PhotoPaint (TM) and several freeware tools will create MPEG movies from such series of files. 
'write a matlab movie variable to jpeg files' Create a series of numbered jpeg files from a Matlab movie variable. This is a special type of variable used by Matlab to generate movies (see 'help movie' for more information). Corel PhotoPaint (TM) and several freeware tools will create MPEG movies from such series of files. 
'Dose Response Curves - script' A script support to generate Dose-Response curves (DRC) based on plots of extracted data. You need to first generate a plot of the dataset you want to sum up in a DRC. The script will then guide you through the steps necessary to compress the plotted data into a DRC. A similar function is available through the 'Average Analysis-Results ...' function in the LFP Tools menu.
Graphics Utilities some tools that, in part, were replaced by the figure editing functions of the figure menues.
- 'Simple Plot Editor' A Matlab function that is useful to edit/annotate figures, obsolete for Matlab version 5.3 and higher.
- 'Print Annotation - script' A script to help you with some common tasks when creating figures.
- 'Subplot Mosaic - script' A script to combine subplots from different figures into one figure, sometimes useful to create presentation figures.
Demos
- 'Cerebellum Movie Demo' A demo movie showing signal spreading across a cerebellar slices. To run this demo you need the files '..\meatools\demodata\Cb090998_e35b.jpg' and '..\meatools\demodata\cb090998b4cont.001'. 
- 'Hippocampus Movie Demo' A demo movie showing signal spreading across a hippocampal organotypic culture. To run this demo you need the files '..\meatools\demodata\271p62_otc_with_sketch.jpg' and '..\meatools\demodata\271p62_3.005'. The dataset and figure were kindly provided by Th. Knott.
