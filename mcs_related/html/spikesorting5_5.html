Spike sorting 5/5 - unsorting
Back to overview

 
Unsorting resets the CLUSTERNUMBER value assigned to a set of spikes (which may be all) to 0 (the default) and adjusts PARSESETUP accordingly.

Indicating a specific name (which must be one already available) and a corresponding cluster number you may add a subpopulation to another.
 
Since this may undo a lot of work be careful.
