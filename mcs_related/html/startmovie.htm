Function synopsis

STARTMOVIE - is a wrapper function for meamovie [MEA/ab]

Function comments

 
%   STARTMOVIE without input parameters opens a file dialog and lets 
%   the user select a file from which a datastrm object is to be 
%   constructed whose first sweep is  subsequently read in and displayed 
%   with meamovie in its default configuration 
% 
%   STARTMOVIE(NAME) where <NAME> is a string does nearly the same but 
%   instead of opening the file dialog a datastrm object is constructed 
%   from the file with the name <NAME> 
% 
%   STARTMOVIE(D) where <D> is a a datastrm object reads the first sweep 
%   of <D> and hands it to meamovie 
% 
%   STARTMOVIE(MAT) where <MAT> is a matrix hands this matrix over to 
%   meamovie without the dependence on any datastrm object. The preferred 
%   format for <MAT> presently is 64 x N. Some other formats (68 x N, 
%   N x 64, N x 68, 8 x 8 x N, N x 8 x 8) may be recognized, but in some 
%   cases (e.g. 8 x 8 x 8) there is no way to guess the correct order. 
% 
%   Further arguments are interpreted as parameter/value-pairs. Presently 
%   there are only two (mutually exclusive) parameters that are handled 
%   by STARTMOVIE itself: 'sweep' and 'time'. They are used to set the 
%   position of the datastrm object to a particular sweep. If 'sweep' is 
%   used, the paramter has to be a sweep number while 'time' requires a 
%   time in milliseconds. If both 'sweep' and 'time' are given, the value 
%   of 'sweep' will be ignored. 
%   Information regarding all other possible parameters can be found in 
%   the help for meamovie. 
% 
%   See also MEAMOVIE, STARTMOVIEGUI, DATASTRM, SETPOS
