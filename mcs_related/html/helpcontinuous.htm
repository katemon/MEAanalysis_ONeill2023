helpcontinuous - help file for 'continuous data' tools

Function comments

 
% This section is intended to facilitate access and handling of continuous 
% datasets recorded with MCRack, i.e. datasets not cut as spikes or 
% segments cut after trigger events. 
% 
% !!!!!!!! IMPORTANT !!!!!!!! 
% Continuous Tools require datastreams that are created by recording 
% continuously in MCRack. In Matlab, you can see whether this data stream 
% exists when you create the datastream object. Suitable data have the 
% streamname 'Electrode Raw Data'. 
% 
% LFP Tools will NOT work on streams named 'Spikes 1' or on streams created 
% by an analyzer tool in MCRack! 
% 
% Since I currently do not work with such records there is only 
% one tool implemented as yet. 
% 
% U. Egert 
% 
% Tools 
% retrieve a segment    retrieve a time slice from the record. 
%                       In the dialog box enter as follows: 
%                       1st line: the path & MCRack filename you wish 
%                          to access OR if you have already processed 
%                          part of it in this session just enter D for 
%                          the current datastream object. 
% 
%                       2nd line: enter 0 to retrieve from the beginning, 
%                          any other number giving the onset of the de- 
%                          sired section in milliseconds. 
% 
%                       3rd line: as 2nd except its the end of the section. 
%                          This value must be greater than the start value. 
% 
%                       4th line: the name of the variable to hold the data. 
%                          This will be a 68xN array with N the number of 
%                          samples in the time slice. The data will be in 
%                          microvolts. 
% 
%                       5th line: enter 'all' (without quotes) to retrieve 
%                          data for all electrodes. Any electrode name 
%                          to retrieve data for that electrodes only. 
%                          Enter a set of electrode names as strings in a 
%                          cell array to retrieve data for several electrodes. 
%                          In this case the resulting array will be MxN with 
%                          M the number of electrodes given. This type does 
%                          NOT fit into any other processing or display routines 
%                          smoothly because the standard electrode sequence is 
%                          not preserved. It is useful when it is convenient 
%                          to save memory space and to process data from one 
%                          electrode at a time. 
% 
% U. Egert 11/99
