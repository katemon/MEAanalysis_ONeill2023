prepare_surf4copy - changes the visibility of vector and bitmap graphics in the figure

Function comments

 
% prepare_surf4copy(sfh,action) changes the visibility of different figure 
% elements to enable to export the vector and bitmap graphics parts 
% separately. 
% The default value of <action> is 'export'. If no argument is given, 
% a file open dialog box is given. If <sfh> is missing, the action will be 
% performed on the current figure. 
% 
% <action> can be one of the following strings: 
% axes      -   surface and image objects will be set to visibility off 
% surface   -   axes, text and line objects will be set to visibility off 
% both      -   everything will be visible again 
% export    -   creates separate wmf-files for the axes lines and 
%               text of a figure and tif-files for the surface/image 
%               objects in these axes. Figures will be saved in default 
%               quality (150 dpi) and in the size they are on the screen. 
%               If the figure consists of several surface objects, each 
%               will be saved separately after its color values where 
%               adapted by BLANK_ZLIM_FOR_ZBUFFER 
% 
% Both files have the same name but different extensions. This utility optimizes 
% the line quality for further image processing. Note that it does not check 
% for duplicate filenames but overwrites duplicates without warning! 
% 
% See also MEASURFPLOT, BLANK_ZLIM_FOR_ZBUFFER, PREPARE_FIG4COPY
