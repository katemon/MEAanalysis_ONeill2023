parsespikesgui - GUI to parse MCRacks spike data files

Function comments

 
% PARSESPIKESGUI will help you to select an MCRacks data file 
% and a directory to use for parsing spike data and extracting 
% the variables SPIKETIME SOURCECHANNELNUMBER PARMAT spkpointer 
% PCVECTORS and some accessory ones. The data will be stored 
% to a file in <storage path>. The filename will be generated 
% from the data file name by dropping the '.' and appending 
% '.mat' to the original name. The parsing process can be 
% interrupted by activating the matlab command window and pressing 
% Ctrl-C. Note that matlab cannot stop while in a mex-file, 
% i.e. while fetching spike cutouts from the disk. Matlab will 
% then stop only after returning to the workspace of the m-file 
% that called the mex-routine. 
% 
% Fields: 
% file name          name and path of the MCRack file 
% storage path       path to use for the results file 
% no. of components  number of principle components stored 
% no. of spikes for PC-calculation: 
%     With records in which a large number of spikes are 
%     found on one electrode calculating the principle components 
%     (PC) can be very memory and time consuming, even with large RAM. 
%     You may reduce this by limiting the number of spikes used to 
%     calculate the principle components, e.g. to 20,000. This will 
%     result in only every n-th spike being included in the calculation. 
%     For example, if you have 100,000 spikes on a channel using 20,000 
%     spikes will result in every 5-th spike used. The PCs derived 
%     are then applied to all spikes. 
% 
% To processing several files at once use the batch-function. 
% 
% U. Egert 6/98
