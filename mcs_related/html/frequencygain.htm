Function synopsis

frequencygain - calculate frequencygain curves from frequency sweeps

Function comments

 
% [DuVmax, DuVmin] = frequencygain(file, varargin) generates a 8x8 subplot 
% diagram with a frequency gain curve for each MEA electrode. The necessary 
% input dataset is a 'triggered sweep' record with a frequency sweep, e.g 
% as generated by a function generator. One of the channels, usually one 
% of the analog channels, must hold a square signal whose duty cycle corresponds 
% to the local sinus frequency. 
% 
% If called without input arguments a dialog will open and request the 
% necessary data 
% 
% varargin may be parameter/value pairs that (a) prevent the dialog box and 
% (b) can be used to override the default settings. 
% 
% Possible PVPs are: 
% 'ttl_e'       channel name for the analog channel with ttl-signal used to calculate the frequencies 
% 'refsweep'    the sweep(s) to use. If several sweep ids are indicated the average of these 
%                 sweeps is used. 
% 'theta'       threshold for the 'high'-level given in µV 
% 'test_e'      channel name to use for a test plot of maxima vs. time 
% 'normalize'   {'yes'} to normalize the data, 'no' for raw data 
% 'yscale'      y-scale to use {'log'} or 'normal'. No log on minima! 
% 'xscale'      x-scale to use {'log'} or 'normal'. 
% 'value'       'max', 'min', {'amp'} 
% 'trange'      'all' or 2-element vector giving time limits in seconds 
% 'meaid'       a string identifying the MEA used. This string is printed 
%                 in supertitles of the figures generated. 
% 
% 'TTLtype'    {'sync'} or 'duty' to indicate if the frequency should be calculated 
%                 from a dataset of sync impulses synchronized to the start 
%                 of each cycle or a dataset of square pulses synchronous to 
%                 the test sine-wave. With 'sync' it is assumed that the last 
%                 cycle is terminated by a pulse. 
% 
% (c) U. Egert 7/99
