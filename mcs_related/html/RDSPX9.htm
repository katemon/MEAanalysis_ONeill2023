% msg = rdspx(infile, outfilepath, varargin) - spike reader V. 9 
% Reads data-files with spike cutouts written with the MCS program 'MCRack' 
% in the versions of 20.4.97 and up. 
% 
% Opens the file <infile> , reads and evaluates header and spike data, calculates 
% spike shape parameters for sorting and stores the data in <outfilepath>. The 
% name of the results file is assigned from <infile>. 
% 
% Parameter/Value pairs that can be modified via <varargin> (examples are the defaults): 
% Parameter           format           function 
% savemem             'yes';           'yes' to avoid swapping with large numbers of spikes on a channel 
% use_npcs            8;               no. of principle components to evaluate 
% UseSpikesForPC      'all';           'all' to calculate PCs from all spikes, any other integer to 
%                                      define otherwise, in which case the spikes 
%                                      1:floor(no_of_spikes/UseSpikesForPC):end will be used; 
% defoutdir           'c:\temp';       default directory for results 
% SpikeCounterStop    1e13;            stop condition during file parsing, useful for debugging with 
%                                      old MCRack versions 
% readprimary         'yes';           find all descriptors, set to 'no' to determine PARMAT only when 
%                                      SPIKETIME and SOURCECHANNELNUMBER are already available 
% clearolddata        'ask';           default action with duplicate global data variables found in the 
%                                      workspace (alternatives: 'cancel', 'clear') 
% pcs                 [];              precalculated principle component vectors, [] if none are used, 
%                                      modifying this will omit the PC-calculation and use the ones given. 
% pcsequence          [];              order of the PCs with respect to the order in RACKSETUP.ChannelID, 
%                                      useful if the electrode sequence has been changed across files. 
% startend            [0; 1e9];        def. time range to evaluate (ms), will be changed in future vers. 
% streamname          'Spikes 1'       def. name of the spike data stream, you can evaluate only one 
%                                      stream at a time. 
% readtrig            'no' 'yes'       read trigger streams 'yes'/'no' 
% correcttrig         'String'         correct for stimulus artifacts 'all'/'none'/'streamname' 
% intv                [ia ie]          interval used for correction in microseconds - e.g. [1000 2000] to delete 
%                                      spikes 1 ms before and 2 ms after the trigger event 
% theta               []               negative threshold for post-recording selection of spikes, ignored if empty, selects spikes smaller than theta
